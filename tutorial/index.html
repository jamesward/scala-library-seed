Project Layout

build.sbt - The sbt build definition

project/build.properties - Defines the sbt version

project/plugins.sbt - The sbt plugin definitions

src/main/scala/com/example/Foo.scala - An example Scala source

src/test/scala/com/example/FooSpec.scala - And example Scala test

Running Tests

Run tests either from the Activator UI or via the command line. From the command line re-run all tests automatically on file change with: 

activator ~test

Publish Library

This project bases its version on information from git. So you will need to initialize a git repo (if you don't already have one): 

git init
Then add your files and commit them: 
git add .
git commit -m init
If your current commit is associated with a git tag then the tag will be used to generate a version number. Otherwise the version is associated with the current commit hash.

The easiest way to publish your library to a public repository is with BinTray. To publish your library to BinTray follow these steps:

  1. Create a BinTray account
  2. In BinTray get your API Key from your profile page
  3. From the command line login to BinTray: 
    activator bintray::changeCredentials
  4. Publish your library: 
    activator publish
